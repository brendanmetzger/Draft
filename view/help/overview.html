A humble todo list; It is both necessary and useful to get an application managed and documented from the beginning. This starting point will exemplify thoses principles.

Checklist

Before you begin, you have created a case-sensitive volume that houses all your web work. You have an Apache 2.4+ installed and a web server running and hosting projects. You have PHP 7.1+ running. You are familiar with virtual-hosting and your hosts file. You have downloaded this framework into the document root of your choosing.

If you are reading this, then obviously you are ready to move forward.

  • Development Environment
  • Choose host and set up dev server

Policies

  • Don't handle anything sensitive
  • Dependencies should be met with the same reaction as a flatulated elevator (Avoid, if unavoidable, make a stinky fug face)
  • Think hard about how data is managed (especially RDBMS and mapping/models/patterns)
  • Never get the easy things wrong (markup)
  • Write very little program code (an entire application need not excede ~1000 SLOC)
