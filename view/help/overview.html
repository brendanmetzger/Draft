A humble todo list to get things up and running. It is both necessary and useful to document projects at the onset. This starting point will exemplify thoses principles.

Checklist

Before you go further, ideally you have created a case-sensitive volume that houses all your web work. You have Apache 2.4+ installed and (local) hosting projects. You have PHP 7.1+ running. You are familiar with virtual-hosting and your hosts file. You have downloaded this framework into the document root of your choosing.

Development Goals

  • Don't handle anything sensitive
  • Dependencies should be met with the same reaction as riding in flatulated elevator (If they unavoidable, crinkle nose and try to blame someone else.)
  • Think hard about how data is managed (especially RDBMS and mapping/models/patterns)
  • Markup is the most important component
  • Write very little program code (an entire application need not excede ~1000 SLOC)
  • No static units in CSS (only proporpotions and relative units)
  • No preprocessors
